# CI/CD Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Set up GitHub repository, CI pipeline, and automated npm publishing via GitHub Actions.

**Architecture:** Two GitHub Actions workflows — `ci.yml` for push/PR testing, `publish.yml` for tag-based npm publish + GitHub Release. Repo created via `gh` CLI, secrets configured manually by the user.

**Tech Stack:** GitHub Actions, pnpm 10, Node 22, npm registry

### Task 2: Create CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

**Step 1: Create the workflow directory and file**

Create `.github/workflows/ci.yml`:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm

      - run: pnpm install --frozen-lockfile
      - run: pnpm run typecheck
      - run: pnpm run test
      - run: pnpm run build
```

**Step 2: Validate YAML syntax**

Run: `node -e "const yaml = require('fs').readFileSync('.github/workflows/ci.yml','utf8'); console.log('YAML file exists, length:', yaml.length)"`
Expected: `YAML file exists, length: <number>`

**Step 3: Commit**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add CI workflow for push/PR to main"
```

### Task 3: Create publish workflow

**Files:**
- Create: `.github/workflows/publish.yml`

**Step 1: Create the publish workflow**

Create `.github/workflows/publish.yml`:

```yaml
name: Publish

on:
  push:
    tags:
      - 'v*'

jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        with:
          version: 10

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
          registry-url: https://registry.npmjs.org

      - run: pnpm install --frozen-lockfile
      - run: pnpm run typecheck
      - run: pnpm run test
      - run: pnpm run build

      - name: Verify tag matches package.json version
        run: |
          TAG_VERSION="${GITHUB_REF#refs/tags/v}"
          PKG_VERSION=$(node -p "require('./package.json').version")
          if [ "$TAG_VERSION" != "$PKG_VERSION" ]; then
            echo "::error::Tag version ($TAG_VERSION) does not match package.json version ($PKG_VERSION)"
            exit 1
          fi
          echo "Version match: $TAG_VERSION"

      - run: npm publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

      - name: Create GitHub Release
        run: gh release create "$GITHUB_REF_NAME" --generate-notes
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**Step 2: Validate YAML syntax**

Run: `node -e "const yaml = require('fs').readFileSync('.github/workflows/publish.yml','utf8'); console.log('YAML file exists, length:', yaml.length)"`
Expected: `YAML file exists, length: <number>`

**Step 3: Commit**

```bash
git add .github/workflows/publish.yml
git commit -m "ci: add publish workflow for tag-based npm release"
```

### Task 6: Manual step — Configure NPM_TOKEN secret

This step CANNOT be automated (secrets must be set via GitHub UI or `gh secret set` with a token value).

**Instructions for the user:**

1. Go to https://www.npmjs.com → Access Tokens → Generate New Token → **Automation** type
2. Copy the token
3. Run: `gh secret set NPM_TOKEN` → paste the token when prompted

After this, tag pushes will be able to publish to npm.

**Step 1: Verify secret is set**

Run: `gh secret list`
Expected: `NPM_TOKEN` appears in the list.
